# What is RAT?

**RAT** is a self-hostable data platform that makes data engineering accessible to everyone. Write SQL (or Python), run pipelines, and query your data â€” all from a web-based IDE.

<Callout type="info">
RAT is built with a Go platform, Python execution engine, and Next.js portal.
Community Edition is free and open-source.
</Callout>

## Why RAT?

- **Anyone can data** â€” If you can write SQL, you can build data pipelines
- **Self-hostable** â€” One `docker compose up` and you're running
- **Batteries included** â€” Editor, scheduler, query console, lineage, quality tests â€” all built-in
- **Apache Iceberg** â€” Your data lives in open table format, not locked in a vendor
- **Git-like isolation** â€” Each pipeline run gets its own branch â€” bad data never reaches production

## How It Works

RAT uses a **medallion architecture** (Bronze â†’ Silver â†’ Gold) to organize data transformations:

<Mermaid chart={`graph LR
    A["Raw Files"] --> B["Bronze\n(raw ingest)"]
    B --> C["Silver\n(cleaned)"]
    C --> D["Gold\n(business-ready)"]
`} />

You write SQL pipelines that transform data between layers. RAT handles execution, scheduling, versioning, and quality testing.

## Quick Example

Here's a complete pipeline that deduplicates raw orders into a clean Silver table:

```sql filename="pipeline.sql"
-- @merge_strategy: incremental
-- @unique_key: order_id
-- @watermark_column: updated_at

SELECT
    order_id,
    customer_id,
    total_amount,
    status,
    updated_at
FROM {{ ref('bronze.raw_orders') }}
{% if is_incremental() %}
WHERE updated_at > '{{ watermark_value }}'
{% endif %}
```
